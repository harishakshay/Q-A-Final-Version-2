# End-to-End Workflow

The **AI Memory & Reasoning Agent** ensures a seamless transition from raw data input to verified, expert-level synthesized outputs.

## 1. Input Layer: Data Acquisition
The system accepts information from two primary sources:
- **Direct Uploads**: Users can upload `.pdf`, `.txt`, and `.md` files via the dashboard.
- **WordPress Integration**: The system can fetch posts directly from specific WordPress APIs, treating them as live memory sources.

## 2. Processing Layer: The Ingestion Pipeline
Once data is received, it undergoes a transformation process:
1. **Conversion**: Non-text documents (like PDFs) are converted into clean, searchable text.
2. **Section-Aware Chunking**: Instead of arbitrary character limits, the `DocumentChunker` respects the document's hierarchy (headings, sub-headings) to preserve context.
3. **Multi-Store Indexing**:
    - **ChromaDB**: Stores semantic vector embeddings for fast similarity search.
    - **Neo4j**: Maps structural relationships, such as document parents, sequential reading order, and shared categories.
4. **Enrichment**: Performance metrics (if available) are merged to flag content health.

## 3. Retrieval Engine: Hybrid Search
When a user asks a question, the system doesn't just look for words; it looks for meaning and connections:
- **Query Classification**: The engine determines if the query requires a structural, semantic, or hybrid retrieval strategy.
- **Vector Retrieval**: Finds the most semantically relevant text fragments from ChromaDB.
- **Graph Expansion**: Neo4j identifies related neighbors or chronological context that the vector search might miss.
- **Multi-Factor Reranking**: All candidates are scored based on semantic distance, graph connectivity, and category metadata.

## 4. Reasoning Layer: LLM Synthesis
The final answer is generated by the **Groq (Llama-3)** reasoning engine:
- **Strict Evidence-Base**: The LLM is restricted to the provided context blocks to prevent hallucination.
- **Dual Format Output**: Every response contains a concise **Direct Answer** and a detailed **Structured Report**.

## 5. Output Layer: Verified Knowledge
The user receives a rich response package:
- **The Answer**: Factual synthesis of retrieved data.
- **Deep Citations**: Precise links to the source document, snippets of the actual text, and confidence scores (High/Medium/Low).
- **Graph Visualization**: Interactive nodes showing how the answer relates to the broader knowledge base.
